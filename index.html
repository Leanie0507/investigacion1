Expresiones Regulares

Introducción a las expresiones regulares en JavaScript

Las expresiones regulares (a menudo llamadas RegExp o RegEx) son un sistema para buscar, capturar o reemplazar texto utilizando patrones. Estos patrones permiten realizar una búsqueda de texto de una forma relativamente sencilla y abstracta, de forma que abarca una gran cantidad de posibilidades que de otra forma sería imposible o muy extensa y compleja. Estos patrones se representan mediante una cadena de texto, donde ciertos símbolos tienen un significado especial.

En JavaScript, las expresiones regulares se escriben entre dos barras diagonales. Ejemplo, /cosa/ busca la palabra "cosa" en un texto. También se puede usar el constructor new RegExp() para crearlas.

Sintaxis de las expresiones regulares en JavaScript

Son letras, números o símbolos que se buscan literalmente en el texto. Por ejemplo, el patrón /abc/ coincide con las combinaciones de caracteres en cadenas solo cuando la secuencia exacta "abc"

Estos tienen un significado especial en las expresiones regulares. Ayudan a definir patrones más avanzados.
  • .: Coincide con cualquier carácter excepto saltos de línea.
  • \d: Coincide con cualquier dígito (0-9).
  • \w: Coincide con cualquier carácter de palabra (letras, números y guion bajo).
  • \s: Coincide con cualquier espacio en blanco.
  • ^: Indica el inicio de la cadena.
  • $: Indica el final de la cadena.

Se combinan caracteres y caracteres especiales para formar patrones complejos.
Ejemplo: /\d{3}-\d{2}-\d{4}/ coincide con un número de seguro social en formato "123-45-6789".

Métodos de JavaScript para trabajar con expresiones regulares

Ejecuta una búsqueda en una cadena y devuelve un array con la primera coincidencia o null si no hay coincidencias.
Ejemplo: /hola/.exec('hola mundo') devuelve ['hola'].

Verifica si una expresión regular encuentra una coincidencia en una cadena, devolviendo true o false.
Ejemplo: /hola/.test('hola mundo') devuelve true.

Devuelve un array con todas las coincidencias encontradas en una cadena.
Ejemplo: 'hola mundo'.match(/o/g) devuelve ['o', 'o'].

Devuelve un iterador con todas las coincidencias y sus grupos de captura.
Ejemplo: 'hola mundo'.matchAll(/o/g) devuelve un iterador con dos coincidencias.

Reemplaza las coincidencias de una expresión regular en una cadena por un nuevo texto.
Ejemplo: 'hola mundo'.replace(/mundo/, 'JavaScript') devuelve 'hola JavaScript'.

Reemplaza todas las coincidencias de una expresión regular en una cadena por un nuevo texto.
Ejemplo: 'hola mundo mundo'.replaceAll(/mundo/g, 'JavaScript') devuelve 'hola JavaScript JavaScript'.

Devuelve el índice de la primera coincidencia de una expresión regular en una cadena o -1 si no hay coincidencias.
Ejemplo: 'hola mundo'.search(/mundo/) devuelve 5.

Divide una cadena en un array de subcadenas utilizando una expresión regular como delimitador.
Ejemplo: 'uno,dos,tres'.split(/,/) devuelve ['uno', 'dos', 'tres'].

Creación de una expresión regular en JavaScript

Se define entre barras diagonales /. Ejemplo: const regexp = /.a.o/i;

Se utiliza la función constructora RegExp para crear una expresión regular.
Ejemplo: // Notación de objeto const regexp = new RegExp(".a.o", "i"); const regexp = new RegExp(/.a.o/, "i");

Rendimiento y consideraciones de seguridad

Las expresiones regulares pueden ser costosas en términos de rendimiento, especialmente si son complejas o se aplican a cadenas grandes. Es importante optimizarlas para evitar problemas de eficiencia.

Las expresiones regulares mal diseñadas pueden ser vulnerables a ataques de denegación de servicio (ReDoS), donde una entrada maliciosa puede hacer que la expresión regular consuma recursos excesivos y bloquee el sistema. Se recomienda validar y sanitizar las entradas del usuario y evitar expresiones regulares que puedan causar este tipo de vulnerabilidades.
